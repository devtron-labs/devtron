App Store Deployment

Update App Store Deployed Application

Updates an existing app store deployed application with new configuration, values, or chart version. This endpoint allows updating Helm applications that were deployed from the Devtron app store.

Key Features:

  • Update application values (values.yaml)
  • Change chart version
  • Update reference values
  • Modify deployment configuration

Authentication:

  • Requires valid user session
  • User must have update permissions for the application
Request Body schema: application/json
required

App store deployment update request containing the new configuration and values for the installed application.

installedAppId
required
integer >= 1

ID of the installed application to update. This is the unique identifier for the deployed app store application.

appStoreVersion
integer >= 1

App store application version ID. Use this to update to a different chart version. If not provided, keeps the current version.

valuesOverrideYaml
string

YAML string containing the values to override the default chart values. This should be valid YAML format.

referenceValueId
required
integer >= 1

Reference value ID for the chart values. This is required and indicates which set of reference values to use as a base.

referenceValueKind
required
string
Enum: "DEFAULT" "TEMPLATE" "DEPLOYED" "EXISTING"

Kind of reference values to use. Must be one of the predefined values.

  • DEFAULT: Use default chart values
  • TEMPLATE: Use template values
  • DEPLOYED: Use currently deployed values
  • EXISTING: Use existing saved values
appName
string <= 100 characters

Name of the application. Optional field for identification purposes.

teamId
integer >= 1

Team/Project ID associated with the application. Optional field.

environmentId
integer >= 1

Environment ID where the application is deployed. Optional field.
