API specification for rotating pods in Devtron applications

App Management

Rotate Pods

Rotates (restarts) pods for a specific application in an environment. This operation triggers a rolling restart of the specified Kubernetes resources.

Use Cases:

  • Restart pods to pick up new configuration changes
  • Force pod recreation for troubleshooting
  • Apply resource updates that require pod restart

Required Permissions:

  • Application trigger permission
  • Environment trigger permission
Authorizations:
bearerAuth
Request Body schema: application/json
required

Pod rotation request containing application details and resources to rotate

appId
required
integer >= 1

Unique identifier of the application

environmentId
required
integer >= 1

Unique identifier of the environment

required
Array of objects (ResourceIdentifier) non-empty

List of Kubernetes resources to rotate
